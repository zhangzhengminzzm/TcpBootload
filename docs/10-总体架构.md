# 总体架构

## 1. 目标

构建一套面向仪表产品的以太网远程升级方案，硬件平台为 `W5500 + STM32F407VGT6`，由上位机通过 `Modbus TCP` 触发并执行 Bootload 升级流程。

## 2. 系统边界

- 下位机（设备侧）：
  - Bootloader（最小通信与写 Flash 能力）
  - Application（正常业务 + 可进入升级模式）
- 上位机（本仓库）：
  - GUI 操作界面（PyQt）
  - 升级任务编排、协议打包、状态展示、日志记录
- 网络：
  - TCP/IP 局域网
  - 单设备升级为主，预留批量升级能力

## 3. 逻辑分层

1. UI 层：设备连接、文件选择、进度与结果展示。
2. 服务层：升级状态机、超时重试、异常恢复。
3. 协议层：Modbus TCP 读写、Bootload 指令封装。
4. 数据层：固件文件解析、分包、校验、日志。

## 4. 核心流程（抽象）

1. 发现设备并建立会话。
2. 校验当前设备状态（版本、运行模式、可升级标志）。
3. 发起进入 Bootloader 命令（通过 Modbus TCP 控制寄存器）。
4. 执行升级握手、擦写、分包下载、整包校验、激活新固件。
5. 设备重启并回读版本确认。

## 5. 可靠性设计要点

- 分包序号与长度双校验，防止错包、乱序包。
- 每包 ACK/NACK，支持失败重传。
- 升级会话级超时和阶段超时分离。
- 断电/中断恢复策略：至少确保旧版本可启动或可重进升级模式。
- 上位机本地完整日志（时间戳 + 阶段 + 原始码）。

## 6. 安全建议（可分阶段）

- 最低阶段：固件 CRC32 + 设备型号匹配校验。
- 进阶阶段：固件签名（RSA/ECDSA）与版本回滚保护。
- 网络隔离：仅在受控网段开放升级端口。
